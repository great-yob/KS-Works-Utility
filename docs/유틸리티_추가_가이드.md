# 새 유틸리티 추가 가이드

KS-Works-Utility는 **포털형** 구조입니다. 유틸 하나 = 사이드바 항목 1개 + 라우트 1개 +
(필요 시) `/api` 라우트 묶음 1개. 아래 순서만 따르면 기존 코드를 건드리지 않고 유틸을
하나씩 붙일 수 있습니다.

---

## 1. 프론트엔드 (필수)

### 1-1. 페이지 컴포넌트 작성
`src/pages/<유틸이름>.tsx` 에 React 컴포넌트를 default export 합니다.
기존 [PdfCompressor.tsx](../src/pages/PdfCompressor.tsx) / [ImageConverter.tsx](../src/pages/ImageConverter.tsx)
를 복사해서 시작하면 됩니다.

- 서버와의 통신은 상대경로 `fetch("/api/...")` 로 합니다.
- 드롭된 파일의 **절대경로**가 필요하면 `window.electronAPI?.getPathForFile(file)` 를 사용합니다.

### 1-2. 레지스트리에 한 줄 등록
[src/modules/registry.tsx](../src/modules/registry.tsx) 의 `modules` 배열에 항목을 추가합니다.

```tsx
import { Scissors } from "lucide-react";          // 아이콘 추가
import PdfMerger from "../pages/PdfMerger";        // 페이지 추가

export const modules: UtilityModule[] = [
  // ...기존 항목...
  {
    id: "pdf-merger",        // kebab-case, 유일값
    path: "/merge",          // 라우트 ("/" 는 단 하나만)
    label: "PDF 병합기",      // 사이드바 라벨
    icon: Scissors,          // lucide-react 아이콘
    accent: "emerald",       // blue | indigo | emerald | amber | rose
    Component: PdfMerger,
  },
];
```

이게 전부입니다. `App.tsx` 는 이 배열만 보고 사이드바와 라우팅을 자동 생성합니다.

> **accent 색상 추가하기**: 새 색상이 필요하면 `src/modules/types.ts` 의 `AccentColor`
> 와 `src/App.tsx` 의 `ACCENT_ACTIVE` 맵 양쪽에 같은 키를 추가하세요. (Tailwind가
> 클래스를 보존하도록 클래스 문자열은 반드시 **리터럴 전체**로 적어야 합니다.)

## 2. 백엔드 (해당 유틸이 서버 작업을 할 때만)

순수 프론트 유틸이면 이 단계는 건너뜁니다.

### 2-1. API 모듈 작성
[api-modules/appInfo.ts](../api-modules/appInfo.ts) 를 복사해 `api-modules/<유틸이름>.ts` 를 만듭니다.

```ts
import type { Express } from "express";
import type { ApiModule } from "./types";
import { tempDir, upload, getResourcePath } from "./shared";

export const pdfMergerModule: ApiModule = {
  id: "pdf-merger",
  register(app: Express) {
    app.post("/api/merge", upload.array("files"), async (req, res) => {
      // ... 처리 후 res.json(...)
    });
  },
};
```

공용 헬퍼는 [api-modules/shared.ts](../api-modules/shared.ts) 에 있습니다.
- `tempDir` — 모든 유틸이 공유하는 임시 폴더
- `upload` — multer 인스턴스 (디스크 저장, 500MB)
- `getResourcePath(...segs)` — 번들된 `resources/` 경로 (dev/패키지 양쪽 자동 처리)
- `getGhostscriptPath()`, `getImageWorkerPath()` — 네이티브 바이너리 경로

### 2-2. 레지스트리에 등록
[api-modules/registry.ts](../api-modules/registry.ts) 의 `apiModules` 배열에 추가합니다.

```ts
import { pdfMergerModule } from "./pdfMerger";

export const apiModules: ApiModule[] = [
  appInfoModule,
  pdfMergerModule,   // 추가
];
```

`server.ts` 가 시작 시 `registerModules(app)` 로 전부 마운트합니다.

> 새 네이티브 바이너리(예: 다른 exe)를 묶어야 하면 `resources/` 아래에 넣고
> `getResourcePath(...)` 로 참조하세요. `resources/` 는 electron-builder의
> `extraResources` 로 asar 바깥에 배포됩니다.

## 3. 체크리스트

- [ ] `src/pages/<이름>.tsx` 생성
- [ ] `src/modules/registry.tsx` 에 항목 1줄 추가
- [ ] (서버 작업 시) `api-modules/<이름>.ts` 생성 + `api-modules/registry.ts` 등록
- [ ] `npm run lint` 통과 (타입 체크)
- [ ] `npm run dev` 로 동작 확인
- [ ] `package.json` 의 `version` 올리고 배포 → [자동업데이트_배포_가이드.md](./자동업데이트_배포_가이드.md)
